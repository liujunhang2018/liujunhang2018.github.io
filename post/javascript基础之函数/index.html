函数课程之前的准备和了解

在来了解函数之前，我们先来看一下下面的这段代码：

<!DOCTYPE html>
<html>
<head>
    <title>示例</title>
    <script type="text/javascript">
        var sum=0;
        for(var i=1;i<50;i+=2)
        {
            sum+=i;
        }
        document.write("50以内所有奇数和为："+sum);
    </script>
</head>
<body>
</body>
</html>

上面代码的功能是计算50以内奇数的和，代码很简单，但是如果我们还需要再来计算下100以内的奇数和呢？

 <script type="text/javascript">
        var sum1=0;
        for(var i=1;i<50;i+=2)
        {
            sum1+=i;
        }
        document.write("50以内所有奇数和为："+sum1);
        document.write("<br/>");
        var sum2=0;
        for(var j=1;j<100;j+=2)
        {
            sum2+=j;
        }
        document.write("100以内所有奇数和为："+sum2);
    </script>

我们只需要将代码进行简单的复制和更改，就可以获得我们想要的答案，但是如果这个时候的需求是，计算50、100、150、220等等数值以内的奇数和，我们该怎么做呢？难道要不断的复制代码吗？

如果真的这样做了，那么我们的代码就会变得异常冗余。

这个时候，我们今天的主角函数就要登场了。

虽然没有学函数，但是我们不妨先来看下效果：

 <script type="text/javascript">
        //定义函数
        function sum(n)
        {
            var m=0;
            for(var i=1;i<n;i+=2)
            {
                m+=i;
            }
            document.write(n+"以内所有奇数和为："+m+"<br/>");
        }
        //调用函数，计算50以内所有奇数的和
        sum(50);
        //调用函数，计算100以内所有奇数的和
        sum(100);
        //调用函数，计算150以内所有奇数的和
        sum(150);
        //调用函数，计算200以内所有奇数的和
        sum(200);
        //调用函数，计算250以内所有奇数的和
        sum(250);
    </script>

在浏览器内的效果如下：

你会发现，通过函数的形式，可以节省大量的工作，而使用函数只需要两步：定义函数、调用函数。

函数的概念

函数，就是一个一系列JavaScript语句的集合，这是为了完成某一个会重复使用的特定功能。在需要该功能的时候，直接调用函数即可，而不必每次都编写一大堆重复的代码。并且在需要修改该功能的时候，也只要修改和维护这一个函数即可。

总之，将语句集合成函数，好处就是方便代码重用。并且，一个好的函数名，可以让人一眼就知道这个函数实现的是什么功能，方便维护。

定义一个函数

我们在上面说过，想要使用函数，需要两个步骤：定义函数、调用函数。

那么该如何定义一个函数呢？

在JavaScript中，可以通过function关键字来定义一个函数(并且function关键字不可以省略)。

Tip：在定义一个函数的时候，你可以选择给函数设置一个名字，也可以选择不给他设置名字。

不设置函数名的函数（赋值式）

例如下面的函数语法就是没有设置名字的函数

function (参数1,参数2...) {
	// 函数体语句
}

示例：

<script type="text/javascript">
		
		var documentFont = function () { // 定义函数。
			document.write("这是一段输出在页面当中的文字");
		}
		documentFont(); // 调用函数

</script>

效果如下：

Tip：定义一个函数之后，函数本身并不会执行，而是需要调用才可以执行。这一点一定要注意。

设置函数名的函数(声明式)

“指定函数名的函数”是JavaScript中使用最广泛的方法，反而“不指定函数名的函数”用得比较少。

语法：

function 函数名(参数1,参数2,….,参数n) 
{
    //函数体语句
    return 表达式;
}

解析：

定义函数必须使用function关键字。

函数名必须是唯一的，尽量通俗易懂，并且跟你定义的代码有关。

函数可以使用return语句将某个值返回，也可以没有返回值。

参数是可选的，可以不带参数，也可以带多个参数。如果是多个参数的话，参数之间要用英文逗号隔开。

示例：

<script type="text/javascript">
	 //定义函数
        function addNum(a,b)
        {
            var m=a+b;
            return m;
        }
        //调用函数
        var sum=addNum(1,2);
        document.write("相加的结果是："+sum);
</script>

结果：

我们在定义函数的时候用到的参数，那就叫“形参”，例如上面的a和b就是形参。我们在调用函数时，与形参对应的数据叫“实参。”其实“形参”和“实参”从名字上面来看就很好理解，所谓的形参就是“形同虚设”而到头来被其他数据替代掉的参数，实参就是实实在在的数据。

使用形参和实参，需要注意形参和实参是一一对应的。通常，在定义函数时使用了多少形参，那么在函数调用时也必须给出多少个参数。

一句话：定义函数时的参数叫“形参”，调用函数时的参数叫“实参”。

函数的调用

使用函数的两个需要注意的问题：定义函数、调用函数。

下面来说一下函数的调用。

常用的函数的调用方式有四种：

  • 简单调用
  • 在表达式中调用
  • 在事件响应中调用
  • 通过链接调用

下面我们分别来说一下不同的调用方式：

简单调用

JS函数简单调用，也被称为直接调用。该方法一般比较适用于没有返回值的函数。也就是说相当于执行函数中的语句集合。

函数名(实参1,实参2,….实参n);

示例：

<script type="text/javascript">
	  //定义函数
        function alertSum()
        {
            //计算1+2+3+…+10的和
            var sum=0;
            for(var i=1;i<=10;i++)
            {
                sum=sum+i;
            }
        alert(sum);
        }
        //调用函数
        alertSum();
</script>

在表达式中调用

在表达式中调用函数的方式，一般适用于有返回值的函数，然后函数的返回值参与表达式的计算。通常该方式还会和输出语句（如document.write()、alert()等）搭配使用。

例如下面的示例，来判断给定的年份是否是闰年：

公历闰年的简单计算方法（符合以下条件之一的年份即为闰年） 1.能被4整除而不能被100整除. 2.能被400整除.

<script type="text/javascript">
	// 定义函数。
	function isLeapYear(year) {


		// 判断闰年的条件
		if ((year%4==0) && (year%100!=0) || (year%400==0)) {
			return year + "年是闰年";
		}else {
			return year + "年不是闰年";
		}


	}
	// 调用函数 
	// 使用一个变量来承接返回值
	var reYear = isLeapYear(2008);
	document.write(reYear);

在事件中调用

JavaScript是基于事件模型的程序语言，页面加载、用户点击、移动光标等都会产生事件。当事件产生时，JavaScript就可以调用某个函数来针对这个事件做出响应。

<button id="btn">点击</button>
<button onclick="alertHello()">点击</button>
<script type="text/javascript">
	
	// 先找到元素
	var mBtn = document.getElementById('btn');

	// 给这个按钮绑定事件
	mBtn.onclick = function () {
		alert("啊，我被点了一下");
	}

	function alertHello() {
		alert("hi man");
	}
</script>

这就是所谓的在事件响应中调用函数，这种方式非常实用。上面实例中的事件是“鼠标的点击事件”。

通过链接调用函数

函数除了可以在事件响应中调用外，还可以通过超链接来调用函数。

<a href="javascript:函数名"></a>

说明：

在超链接中调用函数，指的就是在a标签的href属性中使用“javascript:函数名”来调用函数。当用户点击该超链接时，就会执行调用的相应函数。

示例：

<a href="javascript:alertHello();">点击</a>
<script type="text/javascript">
	function alertHello() {
		alert('Hi,Man!');
	}
</script>

Tip：我们也可以写成下面这样，这样可以去除隐藏锚点。

<a href="javascript:;">点击</a>

变量在函数中的一些变化

什么是作用域？

作用，能做的事。域，领域。 根据词意理解，作用域解释起来就是说作用范围。

全局变量和局部变量

其实，所谓的全局变量和局部变量，只不过是根据变量的范围来给变量进行了一个区分。说到底，无论是全局变量还是局部变量都是变量。

从作用域的角度来说，全局变量指的就是在作用域范围覆盖全局(也就是当前整个文件)，而局部变量就是作用域范围仅限于某个范围内(在函数内部)。

示例：

<script type="text/javascript">
	

	// 声明全局变量 
	var a = "hello,world";

	function fn1() {
		var b = "this is b"; // 这个是局部变量
        // b 的作用域范围仅限于函数内部
        console.log(b);// this is b
	}

	// 我们可以尝试对两个变量进行调用
	console.log(a); // hello,world
	console.log(b); // b is note defined 此时b的调用位置已经超过了b的作用域范围。

</script>

So,函数的出现，除了带来了方便，也带来了一些改变。例如变量分成了全局变量和局部变量。作用域也分成了全局作用域和局部作用域(也叫函数作用域)。

函数内部的全局变量

示例：

<script type="text/javascript">

	function fn1() {
		a = 10 ;  // 在函数中声明变量并未使用var 

		console.log(a); // 10 
	}
	fn1();

	console.log(a); // 10

</script>

在上面的示例中，在函数中声明变量并没有使用var关键字，那么此时这个变量就变成了一个全局变量。

但是这种方式其实是一种不推荐的写法。此种写法容易造成变量的污染。

变量提升

我们上面了解了全局变量和局部变量，局部作用域(函数作用域)和全局作用域。那么下面再来看另外一种在js中非常有意思的情况：

<script type="text/javascript">
		
	console.log(a);
	var a = 10;
	console.log(a);


</script>

上面的代码，第一个会输出undefined，第二个输出为10。

疑问：第一个console.log()语句，输出的是还没有声明的a，为什么会输出一个undefined。

答：原因是js中的变量提升导致。

变量提升即将变量声明提升到它所在作用域的最开始的部分。上面的代码其实在运行的时候相当于变成了下面这个样子：

<script type="text/javascript">
	var a ;  // 将声明语句提升到了当前变量作用域的最上面	
	console.log(a); // 此时a相当于声明但是没有赋值
	a = 10; // 赋值语句原封不动的呆在原地
	console.log(a); // 此时a已经声明并且赋值
</script>

例如再来看一下下面的示例：

<script type="text/javascript">

	var a = "hello"; 

	function fn1() {
		var b = "world";
	}

</script>

上面的代码在我们看不到的地方发生变量提升后是什么样子呢？

<script type="text/javascript">
	var a ; // a 的变量提升被提升到a作用域的最顶端
	a = "hello"; 

	function fn1() {
		var b; // b声明语句被提升到b作用域的最顶端
		b = "world";
	}

</script>

Ok，相信你，现在应该可以理解了变量的提升，那么再来思考一下下面的这种特殊情况：

可以先来看下面的示例。

<script type="text/javascript">
	

	var a = 10;

	function fn1() {
		console.log(a);
	}

	fn1(); // 调用函数

</script>

上面的函数fn1输出结果相信很多同学都可以猜得到，没错，上面的代码会在控制台输出10.

分析：

我们在函数外部声明了全局变量a，那么这个变量在函数中也同样是可以使用的。

上面的示例很简单，那么看下升级版：

<script type="text/javascript">
	

	var a = 10;

	function fn1() {
		console.log(a);
        var a = 20;
        console.log(a);
	}

	fn1(); // 调用函数

</script>

此时函数中的第一个console.log()语句输出结果为？第二个输出结果为？

第一个输出undefined，为什么呢？

因为函数中重新声明了a，那么这个时候a的声明提升到了函数内部的顶部，覆盖了全局变量a。所以此时直接console.log()a这个变量为undefined。

函数提升

上面我们看了在js中的变量提升，可是在js中，除了变量以外，函数同样也存在着提升现象。

例如：

<script type="text/javascript">
	console.log(fn1); // ƒ fn1() {alert(123);}
	function fn1() {alert(123);}
</script>

我们在函数的声明语句的上面打印了函数，结果没有发生报错，而是顺利的打印出了函数，所以我们说，此时函数也同样发生了变量提升。

当然，也存在一些特殊情况，例如我们说到的函数的另外一种声明方式：

<script type="text/javascript">
	console.log(fn1); // undefined
	var fn1 = function () {
		alert(123);
	}
	// fn1();
</script>

此时，我们会发现，第一句console.log()输出的结果变成了undefined，原因在于函数的声明身上。

这种写法的函数没有直接声明，而是将函数本身赋值给了一个变量的身上。而变量的提升，提升的仅是声明的过程。

可以将上面的代码解释为下面的形式：

<script type="text/javascript">
	
	var fn1;
	console.log(fn1); // 此时我们打印的仅是fn1这个声明但是没有赋值的变量，所以为undefined
	var fn1 = function () {
		alert(123);
	}

</script>

函数参数的升级体验

当实参个数超过形参个数

上面我们在讲解函数的时候，讲解过函数参数分为形参、和实参。

<script type="text/javascript">
	// 函数的形参和实参
	function fn1(a,b,c) { // 形参
		console.log(a,b,c);
	}

	fn1(10,20,30);// 实参
</script>

在上面代码中，函数的形参和实参一一对应。

那么如果实参多于形参会怎么样？

<script type="text/javascript">
	// 函数的形参和实参
	function fn1(a,b,c) { // 形参
		console.log(a,b,c);
	}

	fn1(10,20,30,40);// 实参数量多于形参数量
</script>

你会发现，当实参数量多于形参数量时，在浏览器中依然可以运行并且不会报错。

那么我们如果想要获取这个多余的形参该怎么办呢？

<script type="text/javascript">
	// 函数的形参和实参
	function fn1(a,b,c) { // 形参
		console.log(a,b,c);
		// 获取多余的参数可以依靠arguments
		console.log(arguments);
	}

	fn1(10,20,30,40);// 实参数量多于形参数量
</script>

我们将arguments打印出来看下效果：

你可以在这其中看到，函数在调用过程中传递过来的所有的实参。

那么现在我们想要获取传递过来的多余的参数，可以使用arguments关键字参数直接获取。

<script type="text/javascript">
	// 函数的形参和实参
	function fn1(a,b,c) { // 形参
		console.log(a,b,c);
		// 获取多余的参数可以依靠arguments
		console.log(arguments[3]); // 40
	}

	fn1(10,20,30,40);// 实参数量多于形参数量
</script>

当实参个数少于形参个数

示例：

<script type="text/javascript">
	function fn1(a,b,c) {
		console.log(a,b,c); // 10 20 undefined
	}
	fn1(10,20);
</script>

函数的return

在上面我们说到函数构成的时候，我们讲到了return，下面来看一下。

function fn1() {
    return "hello,world";
}

return 又叫返回值，不同于直接在函数中输出或者其他操作，使用return的目的是希望通过return在函数的最后拿到想要的值，在函数外面继续其他操作。

例如，上面的代码，返回了字符串hello,world，想要使用，可以使用变量来进行接收。

function fn1() {
    return "hello,world";
}
// 使用变量接收
var a = fn1(); // fn1() 会让函数调用执行，而函数的return会将值返回到这个函数的外部，这样我们在调用的时候就可以通过变量接收
console.log(a); // hello,world

当然，变量可以省略。

function fn1() {
    return "hello,world";
}
console.log(fn1()); // hello,world

这样也是没有问题的。

Tip：函数只能有一个返回值。返回值后面的代码将不会执行。

function fn1() {
    return "hello,world";
    console.log('你好'); // 这句代码不会执行
}
fn1();

函数的分类

上面我们讲解了函数的基本用法，但是函数其实也存在不同的分类，而所谓的不同分类指的其实就是函数的不同用法，下面我们来简单的讲解一下不同的函数的用法。

闭包函数

在js中，有全局变量和局部变量之分(在函数内部声明 的变量)。

全局 变量和局部变量在上面的课程中，已经说过，这里不再赘述。

示例：

var a = 10 ;
function fn1() {
    console.log(a); // 函数可以随时调用函数外部的全局变量
}
fn1();//10

反之，在函数之外，则没有办法调用函数内部的变量，除非函数内部声明的是全局变量。

var a = 10;
function fn1() {
	console.log(a); // 调用全局变量a
    var x = 10; // 声明局部变量x
    y = 20; // 声明全局变量y
}
fn1();
console.log(x); // err 没有办法调用
console.log(y); // 可以调用到

如何从外部读取局部变量？

出于种种原因，我们有时候需要得到函数内的局部变量。但是，前面已经说过了，正常情况下，这是办不到的，只有通过变通方法才能实现。

那就是在函数的内部，再定义一个函数。

function f1(){

　　　　var n=999;

　　　　function f2(){
　　　　　　alert(n); // 999
　　　　}

　}

在上面的代码中，函数f2就被包括在函数f1内部，这时f1内部的所有局部变量，对f2都是可见的。但是反过来就不行，f2内部的局部变量，对f1就是不可见的。这就是Javascript语言特有的”链式作用域”结构（chain scope），子对象会一级一级地向上寻找所有父对象的变量。所以，父对象的所有变量，对子对象都是可见的，反之则不成立。

既然f2可以读取f1中的局部变量，那么只要把f2作为返回值，我们不就可以在f1外部读取它的内部变量了吗！

function f1(){

　　　　var n=999;

　　　　function f2(){
　　　　　　alert(n); 
　　　　}

　　　　return f2;

　　}

　　var result=f1();

　　result(); // 999

闭包的概念

上面的代码其实就是一个常见的闭包。在很多书籍里关于闭包的定义非常复杂难懂，而在网上的答案也众口不一。

个人的理解是：

闭包就是能够读取其他函数内部变量的函数。

由于在Javascript语言中，只有函数内部的子函数才能读取局部变量，因此可以把闭包简单理解成”定义在一个函数内部的函数“。

所以，在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁。

闭包函数的用途

闭包可以用在许多地方。它的最大用处有两个，一个是前面提到的可以读取函数内部的变量，另一个就是让这些变量的值始终保持在内存中。

怎么来理解这句话呢？请看下面的代码。

function f1(){

　　　　var n=999;

　　　　nAdd=function(){n+=1}

　　　　function f2(){
　　　　　　alert(n);
　　　　}

　　　　return f2;

　　}

　　var result=f1();

　　result(); // 999

　　nAdd();

　　result(); // 1000

在这段代码中，result实际上就是闭包f2函数。它一共运行了两次，第一次的值是999，第二次的值是1000。这证明了，函数f1中的局部变量n一直保存在内存中，并没有在f1调用后被自动清除。

为什么会这样呢？原因就在于f1是f2的父函数，而f2被赋给了一个全局变量，这导致f2始终在内存中，而f2的存在依赖于f1，因此f1也始终在内存中，不会在调用结束后，被垃圾回收机制（garbage collection）回收。

这段代码中另一个值得注意的地方，就是”nAdd=function(){n+=1}“这一行，首先在nAdd前面没有使用var关键字，因此nAdd是一个全局变量，而不是局部变量。其次，nAdd的值是一个匿名函数（anonymous function），而这个匿名函数本身也是一个闭包，所以nAdd相当于是一个setter，可以在函数外部对函数内部的局部变量进行操作。

使用闭包函数的注意点

1）由于闭包会使得函数中的变量都被保存在内存中，内存消耗很大，所以不能滥用闭包，否则会造成网页的性能问题，在IE中可能导致内存泄露。解决方法是，在退出函数之前，将不使用的局部变量全部删除。

2）闭包会在父函数外部，改变父函数内部变量的值。所以，如果你把父函数当作对象（object）使用，把闭包当作它的公用方法（Public Method），把内部变量当作它的私有属性（private value），这时一定要小心，不要随便改变父函数内部变量的值。

回调函数

定义

回调函数是一段可执行的代码段，它作为一个参数传递给其他的代码，其作用是在需要的时候方便调用这段（回调函数）代码。

简单的说，其实就是将一个函数作为参数传递到另外一个函数当中去，在这个函数的某些时刻执行那个作为参数传递过来的函数。

function fn1(parm) {
	alert("hello,world");
    alert("hi!Man");
    parm();
}
function fn2() {
    alert("我是fn2");
}
fn1(fn2);

上面的fn2作为参数传递到了fn里面，当fn1执行到指定位置的时候，就会调用fn2，这就是一个简单的回调函数。

当然，我们在使用回调函数的时候，更多的时候会在满足某些触发条件之后才会执行回调函数。

function fn1(parm) {
	
    for (var i=0;i<10;i++){
        console.log(i);
        if (i==9) {
            parm();
        }
    }
}
function fn2() {
    alert("我是fn2");
}
fn1(fn2);

例如上面的代码，当i=9时，才会执行回调函数。

例如我们常用的事件处理函数，其实也是一个回调函数。

<button id='btn'>点击</button>
<script>
	var oBtn = document.getElementById('btn');
    oBtn.onclick = function() {
        alert(123);
    }
</script>

当button元素发生单击这样一个操作之后，才会去执行事件处理函数，也就是回调函数。

回调函数的优点和缺点

优点：

  • DRY，避免代码重复

  • 可以将通用的逻辑抽象

  • 业务逻辑分离

  • 提高代码的可维护性和可读性

  • 分离专职函数

缺点：

  • 回调函数层级太多，使用困难
  • 大量的回调函数运行， 消耗内存，影响性能。

递归函数

递归函数简单的说，就是在函数内部调用当前函数本身。不过在使用递归函数的时候要小心造成死循环。

所以，在定义递归函数的时候，需要2个必要条件：

  • 首先要包括一个结束递归的条件
  • 其次包括一个递归调用的语句

示例：

function fact(param) {
    
    if (param < 1) {
        return 1 ;
    }else {
        return param * fact(param-1);
    }
    
}
console.log(fact(3)); // 6

练习

斐波那契数列：

介绍： 从数学的角度来讲，斐波那契数列表示的是这样的一个数列：1、1、2、3、5、8、13、21、34……。

从第三个值开始，等于前两个值得和。

示例：

// 例如求第四个值
	// 斐波那契数列
	function fn1(n) {
		if (n <= 2) {
			return 1 ;
		}else {
			return fn1(n-1) + fn1(n-2);
		}
	}

	console.log(fn1(4)); // 3

素数：

素数又称质数，是大于1的自然数，并且只有1和它本身两个因数。

下面来写一个函数，判断一下是否是素数？

function fn1(n) {

		for (var i=2;i<n;i++){

			if (n % i ==0) {
				return false;
			}
		}
		return true;
	}
	console.log(fn1(8)); // false
	console.log(fn1(2)); // true
	console.log(fn1(3)); // true
	console.log(fn1(6)); // false
